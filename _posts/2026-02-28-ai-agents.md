LLMと人間の知能を比べようとするとき、まず思いつく対比がある。

「LLMはステートレスだ。入力を受け取り、推論し、出力する。それだけだ。脳はステートフルで、経験が不可逆に刻まれていく。だから違う」

この直観は、出発点としては正しい。通常の運用ではLLMは推論時に重みが書き換わらない。会話履歴がなければ「さっきの自分」は存在しない。一方、人間は一度見たものを見なかったことにはできない。神経の可塑性は一方通行で、経験は身体に残る。

だが、この対比はしばらく考えると揺らぐ。
現実のLLMは、単体で裸のまま動いているわけではない。マルチターン会話のコンテキスト管理、外部メモリ、ログ、要約機構、ツール呼び出し、評価器、計画モジュール、安全フィルタ──LLMの「外側」に、それを支える仕組みが積み重なっている。この外側の仕組み全体を、ここではハーネスと呼ぶことにする。

LLM単体はステートレスでも、ハーネス込みのシステムは十分にステートフルだ。セッションをまたいで記憶を保持し、過去の出力を参照して行動を変え、環境との相互作用の中で状態を更新し続ける。

## ハーネス＝更新則：一次情報がそのまま「仕様」になっている

現代のコーディングエージェントにおいて「賢さ」を決めるのは、モデルの内部というより、モデルの外側に置かれた次の4点だ。

- **規約注入**：どのテキスト（規約・方針・注意事項）が、どの順序で読み込まれ、どの範囲に効くか（探索順序・上書き・結合）
- **成果物の固定**：要件・計画・テスト計画などが「次の入力」として残り、工程を安定させること
- **観測の制度化**：トレースやログで「何が起きたか」を外部化し、後から最適化できること
- **行為空間の整形**：何を“できる/できない”か（権限・ツール・境界条件）を外側で定義すること

OpenAI Codexの資料は、これらをかなり露骨に「設計の主語」として語る。たとえば “Unrolling the Codex agent loop” は、エージェントを「賢い関数」ではなく**ループする計算**として分解し、推論→ツール→観測→次の推論…の閉ループで、どの情報がコンテキストに積み上がるかを図つきで説明する。[1]

また “Custom instructions with AGENTS.md” は、AGENTS.mdが“百科事典”ではなく「作業の入り口で規約を注入する装置」であり、**探索順序・優先順位・結合規則**（＝更新則の一部）を仕様として明示している。[3]

さらに “Harness engineering” は、人間がやるべき仕事を「環境設計とフィードバックループ」に寄せて言語化し、閉ループ（観測→行為→再観測）が価値生産として回るために、ハーネスがどこまで担うべきかを整理している。[2]

同じ記事には「**見えない知は存在しない**（What Codex can’t see doesn’t exist）」という、境界（ブランケット）の議論に直結する図もある。モデルの内部に知があっても、ハーネスがそれを観測・参照できないなら、更新則に参加できない。[2]

![見えない知は存在しない（外部知をrepoにエンコードせよ）](https://images.ctfassets.net/kftzwdyauwt9/7uWHsJIC6o3uQPsnQ2Avz9/8be3e321892054bd215afb2b250a176a/OAI_Harness_engineering_The_limits_of_agent_knowledge_desktop-light.png)
> 出典: OpenAI, “Harness engineering: leveraging Codex in an agent-first world” [2]

Cookbook側は、この「更新則」をさらに工程として見せる。マルチエージェントのワークフローやトレースUIの例は、タスク分解・handoff・成果物（REQUIREMENTS.mdやTEST_PLAN.md等）と、観測（評価）の外部化が「工程」として設計対象になることを示している。[4]

一方、Anthropic Claude Codeは同じ問題を別の語彙で切るが、結論は近い。
Claude CodeのDocsは、プロジェクト知をまたぐ仕組みとして **CLAUDE.md（人間が書く規約）** と **auto memory（Claude自身が書く運用知）** を明確に分け、ロードされることを仕様として説明する。[5]

さらにClaude Code settingsは、権限やフック等の設定が**スコープと優先順位**で決まることを仕様化しており、これは「規則の注入」が運用ではなく決定手続きになっている例だ。[6]

そしてAnthropicのsandboxing記事は、更新則をもっと露骨に見せる。モデルが何を“できる”か以前に、外側の境界条件（権限・隔離・ゲート）によって**許可された行為空間が整形される**ことが図で示されている。[7]

![Claude Code sandboxing（境界条件で行為空間を整形）](https://www-cdn.anthropic.com/images/4zrzovbb/website/0d1c612947c798aef48e6ab4beb7e8544da9d41a-4096x2305.png)
> 出典: Anthropic, “Making Claude Code more secure and autonomous with sandboxing” [7]

ここまでの具体例が示すのは、ひとつの経験則だ。**価値生産の再現性を決める更新則は、モデル重みの内部よりも、（規約注入・成果物・観測・行為空間）として外在化されがち**である。この外在化が進むほど、議論の焦点は「モデルはステートレスか？」から、「どの境界の内側に、どの更新則を閉じ込めるか？」へ移動する。

## 人間もハーネスを使っている

ここで視点を裏返す。人間は脳だけで知能を完結させているだろうか？

TODOリスト、ノート、スマートフォン、インターネット、言語、法律、制度、教育。これらはすべて「外部に状態を保持し、個体の判断と行動を変える仕組み」だ。つまり、人間もハーネスを使っている。脳が本体で、外部ツールはおまけ──という見方は、実態から乖離している。

この構図は人間に限らない。アリはフェロモンを環境に塗布し、その痕跡が他個体の行動を変える。環境が外部メモリとして機能し、集団全体として探索問題を解く。知能は個体の内部に閉じておらず、個体と環境の結合系に分散している。
（この種の「環境を外部記憶として使う分散的な集団の探索・学習」が理論化されている例として、下記が挙げられる：[「集団の賢さ」を理論で解明](https://www.iis.u-tokyo.ac.jp/ja/news/4889/)）

すると、LLMエージェントと人間（あるいは社会性昆虫）は、どちらも同じ骨格を持つように見えてくる。観測を受け取る。内部状態を更新する。行動する。行動が環境を変え、それが次の観測になる。制御理論で言う閉ループの状態機械だ。この水準で見れば、LLMエージェントも人間も同型である。

## 差分は「更新則」と「価値」の所在にある

比較すべき軸はこうだ。

- どこにstateがあるか
- stateがどう更新されるか（更新則）
- 何が最適化されているか（価値・目的）
- それがどの時間スケールで統合されているか

人間の脳は、推論と学習が密に絡み合っている。何かに注意を向ける、記憶を定着させる、習慣を形成する、危険を回避する──こうした機能が、ミリ秒から年単位まで、多重の時間スケールで自動的に走る。更新則が個体の内部に折り畳まれている。

一方、現在のLLMエージェントでは、これらの多くが外在化されている。行動の一貫性は、メモリ設計が決める。目標の追従は、スケジューラが決める。安全性は、ガードレールが決める。注意の配分や割り込みは、監視モジュールが決める。

つまり、意識や主体性と呼ばれるものの主要成分──目的の維持、注意の制御、自己修正──が、LLM本体ではなく、その外側のハーネスに実装されている。

この事実が意味するのは、比較の正しい構図が「LLM対人間」ではなく、「LLMを核にした全システム対生物個体」だということだ。

## 「価値を認識する」とはどういう条件か

ここから議論をもう一段深くする。主体性や機能意識を論じるなら、その中心に来るのは「価値」だ。

価値を神秘化せずに定義する。価値とは、状態・行為・信念に優先度を与え、注意・記憶・行動の配分と学習の更新方向を決める基準である。

この定義のもとで、あるエージェントが「価値を認識している」と言えるための最低条件を三つ挙げる。

- **条件A**：維持したい量が内部状態として存在する。　たとえば「ユーザーの信頼」「タスクの完了率」「自身の稼働可能性」のような、エージェントにとって保全すべき変数があること。
- **条件B**：その量が注意・記憶・行動を因果的に駆動する。　保全変数がただ存在するだけでは足りない。それが実際にエージェントの振る舞いを変えている必要がある。危険を検知したら別の行動をとる、重要な情報を優先的に記憶する、といった因果の連鎖があること。
- **条件C**：逸脱が起きたとき、再発確率を下げる方向に更新が走る。　これが最も重要で、最も難しい。失敗した瞬間にその場を取り繕うだけではなく、将来の失敗確率を下げる方向に自分自身が変わること。言い換えれば、「行動の修正」ではなく「行動を生む仕組みの修正」が起きること。
条件Cは、制御理論の言葉で言えば二階の閉ループに相当する。一階のループは「行動→結果→行動の調整」。二階のループは「行動を生む方策そのもの→その方策の評価→方策の修正」。人間の学習は、この二階のループを常に回している。

火傷をした子供は、火に近づく行動を避けるだけでなく、「高温のものは危険だ」という方策そのものを獲得する。

現在のLLMエージェントは、この条件をどこまで満たしているか。推論中の自己修正（条件Aに近い）、ツール使用戦略の学習（条件Bに近い）、RLHFによる方策改善（条件Cの一部）は、すでに実現されている。だが、価値や学習則そのものを自己再構成する──条件Cの強い形──は、運用上の安全性と責任の境界の問題から、意図的に外在化されているのが現状だ。

## 最難関：「痛み」を機能として定義する

条件Cをさらに掘ると、避けて通れない問いにぶつかる。

人間は痛い。有限であり、失敗は身体に返る。回復にはコストがかかる。だから学習する動機がある。

AIエージェントはどうか。権限が剥奪されても、予算が削られても、信用が失墜しても、「痛い」のか？

この問いに答えるために、一つ重要な区別を置く。ここで必要なのは、痛みのクオリア（主観的な「痛い」という感覚）ではない。機能意識の議論で問うべきは、痛みが果たしている制御機能の方だ。
機能としての「痛み」は、三つの役割を担っている。

1. 回避行動を強制する。
2. 注意と学習モードを切り替える（通常モードから警戒モードへ）。
3. 失敗の再発を下げる更新に接続される。

したがって、「失敗コストが主体に帰属する」を機能的に言い直すとこうなる。失敗がその主体の将来の行動可能性を実際に狭め、回復に実コストが必要で、その制約が内部の意思決定と更新に因果的に戻ること。

## 大転回：系として見れば、もう条件を満たしているのでは？

現代のAIエージェントは、個々のセッションでは確かに条件Cを満たしていない。だが、系全体として見たらどうか。

ユーザーがエージェントを使い、失敗事例が蓄積され、開発チームがそれを分析し、強化学習やハーネスの改良でモデルが更新される。個々のセッションのLLMは変わらなくても、「LLM＋ハーネス＋開発組織＋ユーザーフィードバック」という系全体は、失敗を内在化し続けている。二階の閉ループは回っている。

この見方を採ると、AIも人間も「拡張主体」として同型に見える。人間だって言語や制度やインターネットという外部ハーネスなしには現在の知能を維持できないのだから、「個体」と「環境」の切り分けは恣意的だ──という議論が成り立つ。

## 「主体」を定義し直す：optimizer（方策を更新する実体）

「主体」「アクター」「エージェント」──こうした語は日常的に使われるが、ここまでの議論を進めるには曖昧すぎる。もっと硬い定義が要る。

ここでは、主体＝optimizerと定義する。optimizerとは、行動を出力するだけでなく、目的関数に沿って自らの方策を更新する実体のことだ。行動するだけなら関数でいい。更新するからoptimizerなのだ。

この定義を導入すると、先ほどの「系として見れば同型」という見方の裏に隠れていた差分が、はっきりと浮かび上がる。

人間（＋外部ハーネス） の場合、更新は基本的に同一の個体に帰属し続ける。スマートフォンを使おうがインターネットを使おうが、「何を学び、何を覚え、何を目指すか」を最終的に決めているのは一人の人間だ。外部ツールは、同一optimizerの手足や記憶の延長として機能する。

現代のAIエージェント系の場合、更新と目的が複数のoptimizerに分裂する。ユーザーの要求、開発組織のポリシー、市場の圧力、規制当局の要件、モデル自体の方策──これらは互いに独立した最適化主体であり、しばしば矛盾する目的を持っている。エージェントの行動は、これらの交渉と妥協の結果として出力される。

つまり、人間の外部ハーネスは「同一optimizerの外部メモリ化」であるのに対し、AIの外部ハーネスは「異なるoptimizerの介入と交渉の場」になりやすい。

ここに、先ほど「主体の境界が不安定」と曖昧に呼んだものの正体がある。

## しかし人間も一枚岩ではない──ここから「統合」が核心になる

ここで当然の反論が来る。

「人間だって一枚岩のoptimizerではないだろう。衝動と理性は矛盾する。短期的快楽と長期的目標は対立する。恐怖反応は意識的判断を無視して走る。人間の内部にも、複数の準最適化ループがあるのでは？」

その通りだ。人間も複数のoptimizerのゲームを内部で走らせている。

だとすれば、人間とAIの差は「optimizerが単一か複数か」ではない。差は統合の度合いにある。

統合とは、0か1かのスイッチではない。連続的な度合いを持つ性質であり、より正確には閉包性（closure）と呼ぶべきものだ。複数の最適化ループが存在するとしても、それらがある境界の内側で強く結合し、価値・学習・コストの帰属が一定の範囲に「閉じて」いるとき、その系は「統合されている」と言える。

人間の場合、衝動と理性が矛盾しても、最終的にはひとつの身体がひとつの行動を選ぶ。痛みも喜びも同一の身体に帰属する。学習の結果は同一の神経系に蓄積される。どれだけ内部で葛藤があっても、価値・学習・コストの閉包が比較的強く個体に帰属し続ける。これが人間の統合の基盤だ。

この「境界」と「閉包性」を数学的に扱う道具として、マルコフブランケット（Markov blanket）がある。

細胞膜が細胞の内と外を統計的に分離するように、ある状態変数の集合を「ブランケット」として切り出すと、内部と外部の間の情報の流れが制約される。人間の身体はひとつのブランケットを構成し、その内部で学習・価値・コストが強く結合している。

![マルコフブランケットの図](/assets/images/2026-02-28-ai-agents/markov-blanket.svg)
> Diagram of a Markov blanket by Laughsinthestocks (CC0, Wikimedia Commons)

同時に、社会や組織もまた、より緩やかなブランケットとして記述できる。ブランケットは入れ子構造を持ち、どのレベルの閉包を「主体」と呼ぶかは、分析の目的に依存する。

## ここまでの道筋を圧縮する

「LLMはステートレス、人間はステートフル」という対比は入口に過ぎない。

ハーネス込みで見れば、LLMエージェントも人間もどちらも閉ループの状態機械であり、外部に状態を持つ点も共通する。だが、差分は消えない。差分は「ステートの有無」から「価値と更新がどこに閉じているか」へ移動する。

人間は、外部ツールを使っても、価値・学習・コストの閉包が比較的強く個体に帰属し続ける（同一optimizerの拡張）。現代のAIエージェント系は、価値と更新が複数のoptimizerに分散しやすく、統合の主体単位が曖昧になる（合成システム）。

したがって、AIに「主体性」や「機能意識」を実装する最難関は、能力の不足ではない。価値と更新の閉包を、ひとつの系の内部にどう作るか──つまり「主体の個体化」（individuation）の問題だ。
